AI Agent Project — Case Study

Clothing Store Outreach Autopilot

An autonomous pipeline that discovers US clothing stores, scrapes their products, selects the best garment using AI, generates a cinematic video, designs a postcard, and physically mails it to the store owner — without human intervention at any step.

01 — How It Works

Why this is an AI Agent, not just automation

Rule-based automation follows fixed scripts: if X do Y. This pipeline makes autonomous decisions — it scores stores, judges garments visually, writes copy, and re-routes when an API fails — all without instructions for each specific case.

01
Discover Stores
Queries OpenStreetMap or Yelp for clothing stores near a target city. Found 56 stores in Los Angeles in under 10 seconds.
OSM / Yelp API
02
Qualify & Score
Scores each store 0–100 on website presence, address completeness, contact info, and social media absence. Filters out chains and incomplete leads. 20 of 56 passed.
Custom scoring
03
Extract Garments
Visits the store's website and scrapes product images using BeautifulSoup. Downloads and caches them locally.
BeautifulSoup
04
AI Picks Best Garment
Sends all images to Groq Vision AI. Scores each on color, silhouette, uniqueness, and cinematic impact. Picks the winner with a written reason.
Groq Llama 4 Scout
05
Generate Video Reel
Optionally runs virtual try-on to place garment on a model (Segmind IDM-VTON). Then animates the image into a 5-second cinematic video (WaveSpeedAI WAN 2.7).
WaveSpeedAI + Segmind
06
Design Postcard
Creates a 4×6" print-ready postcard at 300 DPI. Front: garment image with store-specific headline. Back: personalised letter, QR code linking to the video.
Pillow + qrcode
07
Physically Mail It
Sends to Lob API. Lob prints, addresses, stamps, and delivers it. Returns a tracking ID. Only fires if a video was actually generated — no broken QR codes.
Lob API

What you lose on the free path: try-on model quality (garment shown alone, not on a model), less rich store data, occasional API availability issues.
What you keep: real AI garment selection, real animated video, real physical mail.

04 — What We Tried That Didn't Work

Every combination attempted, why it failed, and how it was resolved or routed around.

What failed Root cause Resolution
HuggingFace Inference API api-inference.huggingface.co blocked by Indian ISP at routing level. DNS change to 8.8.8.8 did not help — it's not a DNS issue, it's a route-level block. Switched to WaveSpeedAI (globally routable)
fal.ai try-on + video storage.fal.run and fal.run both DNS-blocked in India. VPN gave 403 because VPN IP was blacklisted by fal.ai. Abandoned. WaveSpeedAI used instead.
Replicate IDM-VTON + SVD Free trial credits exhausted ($0 balance). 402 Payment Required on every call. Segmind + WaveSpeedAI as free alternative
Segmind IDM-VTON (try-on) 406 Not Acceptable. Payload too large — base64 of 800px images was ~2MB. Server rejected it. Added Pillow resize to 768px before encoding (~200KB). Still getting 406 — under investigation.
WaveSpeedAI with "480p" API only accepts "720p" or "1080p". 400 Bad Request with clear error message. Changed resolution to "720p". Fixed immediately.
WaveSpeedAI connection aborted Sending raw PNG as base64 (~2MB body) caused server to close connection mid-upload. Upload image to tmpfiles.org first, pass HTTPS URL to WaveSpeed. Eliminated payload issue.
WaveSpeedAI with HTTP image URL Garment URL was http:// — WaveSpeed requires HTTPS. 400 Bad Request. Download image locally, re-upload to tmpfiles.org to get HTTPS URL.
Lob API — size "6x4" Lob only accepts "4x6", not "6x4". 422 Unprocessable Entity. Fixed string to "4x6".
Lob API — missing use_type "mail_use_type_can_not_be_null" — required field not sent in payload. Added use_type: "marketing" to request body.
Lob API — wrong address key Code read payload["to"]["city"] but key was "address_city". Sent empty city field. Fixed to read address_parts["city"] directly.
OSM — 0 qualified stores All OSM stores have rating=0, review_count=0. Qualification threshold was 50. Nothing passed. Lower threshold to 15 when no stores have ratings (OSM-specific logic).
Groq 400 — deprecated model llama-3.2-90b-vision-preview removed by Groq with no warning. Switched to meta-llama/llama-4-scout-17b-16e-instruct.
Overpass API 504 Main OSM server overloaded. Returned 504 Gateway Timeout. Added 2 mirror URLs with automatic fallback retry loop.
Postcards mailed with no video Pipeline continued to mail even when video generation failed. QR codes pointed to fake URLs. Gates Steps 6+7 on video_path being set. Stops pipeline and logs warning if no video.
OSM addresses not parseable OSM returns space-separated addresses like "106 East 17th Street Los Angeles CA 90015" instead of comma-separated. Parser failed to split them. Added second parser branch for space-only format with street type detection.
Built and tested against real Los Angeles clothing stores via OpenStreetMap. All pipeline steps are autonomous after a single CLI command. Cost per store on free path: ~$0.06–$0.10.
